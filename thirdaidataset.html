  1. load_bolt_svm_dataset(filename: str, batch_size: int, softmax_for_multiclass: bool = True) -> Tuple[thirdai._thirdai.dataset.BoltDataset, thirdai._thirdai.dataset.BoltDataset]

Loads a BoltDataset from an SVM file. Each line in the input file represents a sparse input vector and should follow this format: ` l_0,l_1,...,l_m i_0:v_0 i_1:v_1 ...     i_n:v_n ` where l_0,l_1,…,l_m is an arbitrary number of categorical labels (integers only), and each i:v is an index-value pair representing a non-zero element in the vector. There can be an arbitrary number of these index-value pairs.

Parameters:

  • filename (*) – String - Path to input file.

  • batch_size (*) – Int (positive) - Size of each batch in the dataset.

  • softmax_for_multiclass (*) – Bool (default is true) - Multi-label samples must be processed slightly differently if softmax is being used in the output layer instead of sigmoid. When this flag is true the loader will process samples with multiple labels assuming that softmax and CategoricalCrossEntropy are being used for multi-label datasets. If the dataset is single label, then this argument has no effect.

Returns a tuple containing a BoltDataset to store the data itself, and a BoltDataset storing the labels.

thirdai.dataset.load_click_through_dataset(filename: str, batch_size: int, max_num_numerical_features: int, max_categorical_features: int, delimiter: str = '\t') → Tuple[thirdai::dataset::InMemoryDataset<thirdai::BoltBatch>, thirdai::dataset::InMemoryDataset<thirdai::BoltBatch>, thirdai::dataset::InMemoryDataset<thirdai::BoltBatch>]

Loads a Clickthrough dataset from a file. To be used with DLRM. Each line of the input file should follow this format: ` l       d_1     d_2     ...     d_m     c_1     c_2     ...     c_n` where l is the label, d is a numerical (quantitative) feature, m is the expected number of numerical features, c is a categorical feature (integer only), and n is the expected number of categorical features.

Parameters:

  • filename (*) – String - Path to input file.

  • batch_size (*) – Int (positive) - Size of each batch in the dataset.

  • num_numerical_features (*) – Int (positive) - Number of expected numerical features in each dataset.

  • max_categorical_features (*) – Int (positive) - Maximum number of expected categorical features in each dataset.

Returns a tuple containing a BoltDataset, BoltTokenDataset to store the dense and categorical features, and a BoltDataset storing the labels.
